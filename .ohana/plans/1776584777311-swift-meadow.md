# Plan: Mradi Africa Website Redesign

## Summary
Rewrite the Mradi Africa website to match the Forge Studio design template. Use CSS classes from the template in `global.css`, create Qwik components for each section, and rebrand with Mradi Africa content showcasing **only realistic projects** (Dairy HQ and Hesabu as real case studies).

## Design Principles (inspired by PodiiHQ analysis)
- **Blue accent palette** — Replace terracotta with professional steel blue (`#2B6CB0`) and its tints
- **Tech stack marquee** — Auto-scrolling horizontal ticker strip between Hero and Projects showing tools/frameworks (Next.js, Go, React, PostgreSQL, Flutter, Python, etc.)
- **Gradient section transitions** — Use `transparent → var(--bg)` fades between sections for smooth visual flow instead of hard edges
- **Generous vertical spacing** — `py-24` / `py-32` between sections for a premium, spacious feel
- **Card hover lift** — `-translate-y-2` with smooth transition on all interactive cards
- **Single cohesive type system** — Syne (headings) + DM Sans (body) used consistently throughout

## Current State
- **Framework**: Qwik v1.19.0 + Qwik City, Vite 7.3.1, Tailwind 3.4.0, Fastify 5.7.4
- **Package manager**: Already migrated to bun (bun.lock exists)
- **CSS variables**: Already in place in global.css matching template tokens
- **Tailwind config**: Already simplified with design token colors
- **Dockerfile**: Already updated for bun
- **Components**: Only `router-head` exists; everything else is monolithic in `index.tsx`

## Current Code Structure Problem
The entire landing page (nav, hero, process steps, CTA, footer — 463 lines) is crammed into a single `src/routes/index.tsx`. There is no component separation. The structure today:

```
src/
├── components/
│   └── router-head/router-head.tsx   ← only component (renders <head> meta)
├── routes/
│   ├── index.tsx                     ← 463-line monolith (ALL sections inline)
│   ├── layout.tsx                    ← minimal: <main><Slot /></main> + cache headers + useServerTimeLoader
│   └── service-worker.ts
├── global.css                        ← old terracotta palette (--accent: #C84B2F)
├── root.tsx                          ← QwikCityProvider shell (imports global.css, RouterHead, RouterOutlet)
├── entry.dev.tsx / entry.ssr.tsx / entry.fastify.tsx / entry.preview.tsx
└── plugins/fastify-qwik.ts
```

**What needs to change:**
- `index.tsx` gets gutted — becomes a clean composition of 10 component imports
- `global.css` gets rewritten — new palette + all template CSS classes
- 10 new component files created under `src/components/`
- `layout.tsx` stays minimal (keep cache headers + `useServerTimeLoader`, remove `<main>` wrapper since components handle their own layout)
- `root.tsx`, entry files, `router-head` — **untouched**

## Implementation Steps

### Step 1: Update `src/global.css` with ALL template styles
Port the **entire CSS** from the template into global.css — nav, hero, marquee ticker, projects grid, project cards, case study modal, services, process steps, about, testimonials, contact form, footer, gradient transitions, and responsive breakpoints at 860px and 560px. This is critical because the template uses specific CSS class names (`.nav-inner`, `.hero`, `.hero-eyebrow`, `.project-card`, `.modal-overlay`, `.marquee-track`, etc.) that the Qwik components will reference.

**Updated color tokens:**
```css
--accent: #2B6CB0;        /* steel blue (was #C84B2F terracotta) */
--accent-hover: #22578A;  /* darker blue for hover states (NEW) */
--accent-light: #E8F0FA;  /* light blue tint (was #F5EAE6) */
--accent-muted: #6B9DD6;  /* softer blue for secondary elements (NEW) */
--border: rgba(20,22,26,0.10);   /* slightly cooler */
--border-mid: rgba(20,22,26,0.16); /* slightly cooler */
```

**New CSS additions (PodiiHQ-inspired):**
- `.marquee-track` — `display: flex; white-space: nowrap; animation: scroll 30s linear infinite;` with `@keyframes scroll { from { transform: translateX(0) } to { transform: translateX(-50%) } }`
- `.section-fade` — gradient overlay class for smooth section transitions
- All sections use `padding: 6rem 0` (py-24) or `padding: 8rem 0` (py-32) for generous spacing
- Card hover: `transition: transform 0.3s ease, box-shadow 0.3s ease; &:hover { transform: translateY(-8px); }`
